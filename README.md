# micro-ayx-resource

本仓库用于归档和发布多个独立的 HTML 页面。这些页面内容相互独立，不针对任何特定网站或域名，仅作为静态资源集合进行维护。

## 项目简介

micro-ayx-resource 是一个轻量级的 HTML 页面归档仓库。其目的是以结构化方式保存和管理一系列独立的 HTML 文件，便于后续查阅、分发或引用。仓库不包含后端逻辑或数据库，所有内容均为纯前端静态页面。

## 目录说明

```
.
├── pages/          # 存放各个独立 HTML 页面
├── assets/         # 公共资源文件（CSS、JS、图片等）
├── index.html      # 仓库首页（可选）
└── README.md       # 本文件
```

- **pages/**：每个子目录或文件对应一个独立的 HTML 页面，命名清晰，便于识别。
- **assets/**：存放页面共享的样式、脚本或媒体文件，避免重复引用。
- **index.html**：可选的入口页面，用于展示页面列表或导航。

## 页面归档说明

- 所有页面均为静态 HTML，不依赖外部服务或 API。
- 页面之间无耦合关系，可单独打开使用。
- 归档时尽量保持文件结构简洁，避免嵌套过深。
- 每个页面建议附带简短说明（可在页面内注释或另附说明文件）。

## 维护说明

- 新增页面时，请放入 `pages/` 目录，并确保资源路径正确。
- 修改现有页面时，请保持原有文件名不变，避免破坏外部引用。
- 不建议在页面中嵌入跟踪代码、广告或动态内容。
- 如需删除页面，请确认无其他页面引用后再移除。

## 使用方式

1. 克隆仓库到本地。
2. 直接在浏览器中打开 `pages/` 下的 HTML 文件，或通过本地 HTTP 服务器预览。
3. 也可将仓库部署到任意静态托管服务（如 GitHub Pages）进行在线访问。
